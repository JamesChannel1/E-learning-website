Diary input

There are three tests. The first test is spelling numbers as words, the second test is a true or false spelling test and the third is writing out the names of the images.

Test 1 summary

Flow of the javascript:

Get element by ID ---> There is an addeventlistener command attached to the button ---> button clicked ---> runs function.

Inside of each function there are variable containers and an if statement. The first variable container stores what the user will input into the text box and then the second variable container stores the correct answer.

The if statement after this is checking if the value the user inputted is equal to the correct answer. If it is the same, then “GetElementByID” command is used to locate the related button element, and then “.innerHTML” is used to output a string “correct! you win”.

If the user enters a value that is not equal to the stored correct answer, then the “else if” part of the function runs, which outputs the string “Incorrect, try again”.

For both, there is a code command for changing the colour to red or green, depending on if it is right or wrong. Due to the button element being selected, this will change the colour of the button along with the text being changed.

Test 2 summary

Flow of the javascript:

Get element by ID ---> There is an addeventlistener command attached to the button ---> button clicked ---> runs function.

This is similar to test 1 except with a "p" html tag being selected as the element where the string from “GetElementByID().innerHTML” is outputted. This means that when the user enters their answer and then clicks the button, text will appear showing if the answer is correct or incorrect.

The HTML on this test is again similar to test 1, with an "h2" heading tag for the title and a "p" tag where I wrote a short description of the test for the user.

Test 3 summary

Flow of the javascript:

Get element by ID ---> There is an addeventlistener command attached to the button ---> button clicked ---> runs function.

This test is also similar to test 1 and test 2 except with images which are added in through HTML and adjusted in CSS. I centered the images and changed their sizes to make them all the same size.

Another difference is when the outputted string is displayed to the user it is inside a "h4" tag. I think this makes it stand out more and looks good.

HTML & CSS

Width is set to 100% for mobile devices, the logo and images are centered, there is a navbar with list items which link to different pages, and there are five main boxes wherein the website content is displayed.

I added a stylesheet link from W3schools to html for the social media icons in the footer. Font is times new roman.
